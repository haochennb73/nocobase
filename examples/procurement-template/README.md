# 採購供應商管理 Excel テンプレート
## Procurement Supplier Management Excel Template

---

### 目的 / Purpose

| 目的 | 詳細 |
|------|------|
| 🗂 供應商名録の収集 | 人力採購に関わるすべての供應商を一覧化 |
| 📊 採購実績の収集 | 契約金額・人数・期間などの実績データを蓄積 |
| 🔍 採購プロセスの把握 | 現行の需要発信・議価プロセスを可視化 |
| 🎯 最終目標 | 採購窓口の統一（単価交渉）、プロセス標準化、コスト削減 |

### サプライヤーシートの構成 / Supplier Sheet Structure

各サプライヤーに 1 シートを割り当て、以下の 3 セクションに情報を記録します。

#### ① 実績情報 (Performance Information)

| フィールド | 説明 |
|-----------|------|
| 年月 (Year/Month) | 取引が発生した年月 |
| 金額 (Amount) | 取引金額（税抜き） |
| 人数 (Headcount) | 従事人員数 |
| 期間 (Period) | 作業期間 |
| BU | 発注元のビジネスユニット |
| 项目内容 (Project) | プロジェクト・作業の概要 |
| 対象客戶 (Client) | エンドクライアント名 |
| 単価区分 (Unit Price) | 適用単価の区分 |

#### ② 調査情報 (Survey Information)

**2-1 需要発信方式**

| フィールド | 説明 |
|-----------|------|
| 時効要求 / Timeliness | 発注から納品までに必要なリードタイム |
| 前倒し量 / Lead Time | 事前通知に必要な期間 |

**2-2 議価プロセス**

| フィールド | 説明 |
|-----------|------|
| 判断要素 | 取引先選定の評価軸（品質・価格・実績等） |
| 競争入札の有無 | 見積もり競合・入札の実施有無 |
| 比較・見積もり | 複数社比較の有無 |
| 価格交渉 | 交渉方式・頻度・担当者 |
| 近年の価格上昇 | 直近 2〜3 年における単価変動 |

**2-3 窓口統一化への懸念**

採購窓口を一本化した場合の懸念・要望を自由記述。

#### ③ 補助情報 (Auxiliary Information)

| フィールド | 説明 |
|-----------|------|
| 会社背景 | 設立年・資本金・主要サービス等 |
| 連絡窓口 | 担当者名・メール・電話番号 |
| 取引年数 | 初回取引開始からの年数 |
| BU長評価 | BU責任者の総合評価（◎/○/△/×）＋コメント |
| その他補足事項 | 特記事項など |

### 使い方 / How to Use

#### テンプレートを直接使用する場合

1. `procurement-supplier-template.xlsx` を開く
2. `📝 新規テンプレート` シートを右クリック →「シートのコピー」を選択
3. コピーしたシートの名前をサプライヤー名に変更する
4. 各セクションに情報を記入する

#### スクリプトからテンプレートを再生成する場合

```bash
# 依存パッケージをインストール（exceljs が必要）
npm install exceljs

# テンプレートを生成
node generate-template.js
```

> **注意**: `generate-template.js` は `exceljs` パッケージを使用します。
> NocoBase のモノレポ内ですでに `exceljs` が利用可能です
> (`packages/plugins/@nocobase/plugin-action-import`)。

### 活用シナリオ / Usage Scenarios

- **初期ヒアリング**: 各 BU の採購担当者に本テンプレートを配布し、既存サプライヤー情報を収集
- **定期更新**: 四半期ごとに実績情報を追記し、採購コストのトレンドを可視化
- **窓口統一化検討**: 調査情報セクションを活用し、供應商ごとの懸念点を整理
- **NocoBase 取り込み**: `plugin-action-import` を使用して、本テンプレートのデータを NocoBase データベースにインポート可能
